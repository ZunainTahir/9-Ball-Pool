# 🎱 9-Ball Pool Game

A fully playable browser-based billiards game built in **C++** and compiled to **WebAssembly** using Emscripten. All game logic — physics, rules, scoring, collisions — runs in C++. The frontend is handled with HTML5 Canvas, CSS, and JavaScript.

## 🎮 How to Play

1. Open the game in Chrome/Edge using a local server (see [Running the Game](#-running-the-game))
2. Type your name and click **Break & Play** (or press `Enter`)
3. **Click and drag** from the cue ball to aim your shot
4. **Release** the mouse to shoot
5. Pot all **8 colored balls first**, then pot the **PINK ball last** to win
6. Pot the pink ball early → **instant LOSS**
7. Pot the cue ball (white) → **FOUL** (−10 seconds)
8. Finish as fast as possible for a higher score!

### Controls
| Action | Control |
|--------|---------|
| Aim | Click and drag from the cue ball |
| Shoot | Release the mouse button |
| Exit game | Click the EXIT button |
| Play again | Click PLAY AGAIN after game ends |

## 📐 Game Rules

- 9 colored balls + 1 white cue ball on the table
- Pot all **8 colored balls** before the pink one
- Potting the **pink ball early** = instant loss
- Potting the **cue ball** = foul (−10 seconds added to elapsed time)
- You have **5 minutes (300 seconds)** to clear the table
- Time runs out = you lose

### 🏆 Scoring Formula
```
Score = WinBonus(300) + TimeBonus(up to 200) + PotBonus(20 × balls potted) − Shots + 100
```
Maximum possible score ≈ **600 points** (fast clear, few shots)

## ⚙️ Setting Up Emscripten (First Time Only)

Open a terminal (PowerShell on Windows, bash on Linux/macOS):

```bash
# 1. Clone the Emscripten SDK
git clone https://github.com/emscripten-core/emsdk.git
cd emsdk

# 2. Install and activate
.\emsdk install latest      # Windows (PowerShell)
.\emsdk activate latest
```

> **SSL error during install?** Download the Node zip manually from the URL shown in the error and place it in `emsdk/zips/`, then re-run `.\emsdk install latest`.

## 🔨 Compiling the Code

> Run this **every time you edit `PoolGame.cpp`**.

```powershell
# Step 1 — Add Emscripten to PATH (run once per terminal session)
$env:PATH = "C:\path\to\emsdk\upstream\emscripten;" + $env:PATH
$env:PATH = "C:\path\to\emsdk\python\3.13.3_64bit;" + $env:PATH

# Step 2 — Go to your project folder
cd "C:\path\to\9-Ball-Pool"

# Step 3 — Compile
em++ -std=c++17 PoolGame.cpp -o PoolGame.js `
  -s EXPORTED_FUNCTIONS="[""_startGame""]" `
  -s EXPORTED_RUNTIME_METHODS="[""ccall""]" `
  -s ALLOW_MEMORY_GROWTH=1
```

After compiling, two files will be created or updated:
- `PoolGame.js`
- `PoolGame.wasm`

## 🚀 Running the Game

> ⚠️ You **cannot** open `index.html` by double-clicking it. WebAssembly requires a local HTTP server.

### Method A — Python HTTP Server (Recommended)

```bash
cd "C:\path\to\9-Ball-Pool"
python -m http.server 8000
```

Then open **http://localhost:8000** in Chrome or Edge.  
Press `Ctrl + C` in the terminal to stop the server.

### Method B — VS Code Live Server

1. Open the project folder in VS Code
2. Right-click `index.html` in the Explorer panel
3. Click **"Open with Live Server"**
4. Game opens at `http://127.0.0.1:5500/index.html`

## 🧱 OOP Concepts Implemented

This project was built as an OOP assignment. Here's how each concept maps to the code:

| Concept | Implementation |
|---------|---------------|
| **Classes & Objects** | `Ball`, `CueBall`, `Hole`, `Physics`, `Rules`, `ScoreManager`, `CanvasRenderer`, `Game` |
| **Encapsulation** | All fields (`x`, `y`, `velocity`, `score`, `timer`) are `private` with getters/setters |
| **Inheritance** | `GameObject → Ball → CueBall` and `GameObject → Hole` and `Renderer → CanvasRenderer` (3-level hierarchy) |
| **Polymorphism** | `draw()` overridden in `Ball`, `CueBall`, `Hole`; `drawGame()` and `drawGameOver()` overridden in `CanvasRenderer` |
| **Abstraction** | `GameObject` and `Renderer` are abstract base classes with pure virtual functions |
| **Composition** | `Game` owns `Physics`, `Rules`, `ScoreManager`, `Renderer` — they are created inside `Game` and destroyed with it |
| **Aggregation** | `Game` manages `balls[]` and `holes[]` as dynamic arrays of independent objects |
| **File Handling** | `ScoreManager` saves and reads scores; file is downloaded to the user's machine at game end |

## 🐛 Troubleshooting

| Problem | Fix |
|---------|-----|
| Black screen after entering name | Recompile `PoolGame.cpp`, then press `Ctrl+Shift+R` in Chrome to hard refresh |
| `em++` not recognized | Re-run the two `$env:PATH` commands in your terminal (required every new session) |
| `Module.ccall is not a function` | `PoolGame.js` has a syntax error — recompile |
| Sounds not playing | Ensure `hit.mp3`, `pot.mp3`, `music.mp3` are in the same folder; click anywhere on the page first (browsers require user interaction before playing audio) |
| `404 wasm file not found` | Make sure `PoolGame.wasm` is in the same folder as `index.html` |
| Can't open by double-click | Always use `python -m http.server` or Live Server — WebAssembly won't load from the filesystem directly |
